# 后端风险登记表

## 1. 高风险项

### 风险 1：本地 JDK 版本过低

- 现状：截至 `2026-03-21`，当前机器检测到 `Java 1.8.0_131`
- 影响：无法构建 `Spring Boot 3.x`
- 处理：优先安装并切换 `JDK 21`

### 风险 2：审核快照和生效数据未分离

- 影响：审核中会直接污染已发布数据
- 后果：审核回溯困难，驳回后无法准确恢复
- 处理：企业、产品都采用“生效数据 + 提交快照”双层建模

### 风险 3：企业数据隔离只做前端控制

- 影响：存在越权读写、导出、下载风险
- 处理：所有企业端接口统一走 `enterprise_id` 后端隔离

### 风险 4：导入走同步接口

- 影响：大文件场景下容易超时和脏数据
- 处理：导入校验与导入执行拆为任务模型

### 风险 5：文件本地存储缺少访问控制

- 影响：证照、图片、PDF 可能被直接遍历访问
- 处理：所有预览和下载都走受控接口，物理路径不外露

### 风险 6：审计日志缺失

- 影响：平台审核、导出、冻结、下架无法追踪
- 处理：建立统一审计与状态变更日志

### 风险 7：质量工具过早过重

- 影响：开发初期规则噪音过大，影响推进速度
- 处理：一期只启用核心质量插件，`PMD / Sonar` 后置

### 风险 8：前后端接口语义漂移

- 影响：联调期间会反复返工
- 处理：以前端 contract + OpenAPI 双基线并行收口

## 2. 中风险项

### 风险 9：行业、类目、单位等字典尚未最终定稿

- 处理：一期使用占位字典，并将字典全部外置

### 风险 10：短信服务与 AI Provider 暂无真实接入

- 处理：先定义 provider 接口与 mock 实现，不阻塞主流程

### 风险 11：导出文件生命周期未定义

- 处理：为导出任务增加过期时间和清理策略

## 3. 当前首要风险顺序

1. `JDK 21` 未安装
2. 审核快照建模
3. 企业数据隔离
4. 审计与导入任务模型
